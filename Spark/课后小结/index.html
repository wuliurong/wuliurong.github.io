一、Spark架构与原理

1、Spark的特点

基于内存的、分布式的

迅速：内存缓存，数据只加载一次

通用：spark SQL、spark streaming、MLib、GraphX

易用：支持多种语言API，本身用scala写的

支持多种资源管理器：使用Hadoop的YARN，，处理HDFS或Hbase数据；支持多种运行模式

2、Spark与Hadoop的区别

Spark作业间是基于内存通信Hadoop作业间是基于磁盘通信

Hadoop是分布式数据设施，将巨大数据储存在由普通计算机组成的集群中Spark则是一个专门的数据处理工具，它不会进行分布式数据的存储Hadoop使用MapReduce进行数据处理Spark处理的数据可存在Hadoop的HDFS或Hbase中

3、Spark生态系统组件及基本用途

Spark SQL：SQL语句转成RDD，操作结构化数据的组件

Spark streaming：实时流计算框架（storm）

MLib：机器学习库，提供很多机器学习的算法

GraphX：图表计算，分布式图形处理框架

Spark core：内核，相当于mapreduce，核心为RDD

4、Spark的应用场景

保险行业：通过使用Spark的机器学习功能来处理和分析所有索赔，优化索赔报销流程

零售业：使用Spark分析销售点数据和优惠券使用情况。互联网：使用Spark的ML功能来识别虚假的配置文件，并增强他们向客户展示的产品匹配。银行业：使用机器学习模型来预测某些金融产品的零售银行客户的资料。科学研究：通过时间，深度，地理分析地震事件来预测未来的事件。地理空间分析：按时间和地理分析Uber旅行，以预测未来的需求和定价。Twitter情绪分析：分析大量的推文，以确定特定组织和产品的积极，消极或中立的情绪。医疗保健：使用Spark Core，Streaming和SQL构建病人护理系统。

5、Spark架构，各节点职责

Driver Program：客户端驱动程序，用于执行Spark任务，主要负责：将用户程序转换为作业在Executor间调度任务跟踪Executor的执行情况通过特定UI查询运行情况

Cluster Manager：Spark的集群管理器，主要负责资源的分配与管理目前，Standalone、YARN、Mesos、EC2等都可以作为Spark的集群管理器 Worker：Spark的工作节点，一个Worker执行在一台服务器上，主要负责：创建Executor（执行器）将资源和任务分配给Executor同步资源信息给Cluster Manager

Executor：执行计算任务的一个进程，主要负责：运行各任务（Task），并将结果返回给驱动器让任务可以利用缓存数据加速计算

6、Spark作业运行流程

二、Spark RDD

1、RDD的概念、内容、运行流程

弹性分布式数据集（RDD），是只读的、分区记录的集合，只能基于在稳定物理存储中的数据集和其他已有的RDD上执行确定性操作来创建1、弹性： 自动切换存储方式，优先存内存，内存不足再存磁盘 RDD可以存储任意类型、可大可小的数据2、分布式： 有时数据量很大，需分布在不同机器中计算3、数据集：RDD是一个抽象数据集，不存储真实数据，只存储一些描述信息初代RDD：只存储真实数据的分区信息等子代RDD：存储描述“初代RDD做了什么操作得到自己”4、只读 RDD是不可变的，如果需要在一个RDD上进行转换操作，则会生成一个新的RDD5、分区记录 RDD是分区的，RDD里面的抽象数据是分布在多台机器上的Executor里面的6、什么时候创建根据文件系统中的指定数据集创建（初代RDD）由其他RDD经过一定操作得到（子代RDD）

运行流程：

RDD执行任务流程1、从文件系统中选择想要计算的数据，创建第一个RDD，此RDD记录数据的位置、分区等信息2、根据需要进行一系列转换，如排序、过滤、去重、map等，每进行一次转换，创建一个新RDD，每个RDD仅记录下以下信息：谁传过来的，我要执行什么操作，我要传给谁3、所有转换完成后，发起执行命令，RDD开始创建任务，读取数据，依次执行所有计算，得出结果RDD总结 相当于一个代理，用户需要进行什么操作，代理先全部记录下来，当用户需要结果时，代理再通知工人们执行任务

2、RDD的创建方式

1、基于外部数据源创建rdd = sc.textFile(“路径”)rdd.count()sc是一个SparkContext对象，返回的rdd对应一个文本文件，count函数可以获取文本的行数 2、数据集合创建data = [1, 2, 3, 4, 5]rdd = sc.parallelize(data) #通过并行化创建RDDrdd.count()rdd对应一个数据集合（如列表），count函数可以获取元素个数

3、转换算子与执行算子区分

RDD的操作1、转换算子：在一个已存在的 RDD上创建一个新的 RDD，但实际的计算并没有执行，仅仅记录操作过程2、行动算子：执行 RDD记录的所有转换操作，并计算结果 转换算子一定会返回一个RDD，行动算子返回计算结果，也可以没有返回值

4、常用转换算子及含义、用法（一维类、key-value类）

1、map：接受一个函数作为参数，对每个元素执行这个函数的转换示例：rdd = sc.parallelize([“b”, “a”, “c”]) sorted(rdd.map(lambda x: (x,1)).collect()) 结果：[(‘a’, 1), (‘b’, 1), (‘c’, 1)]

2、flatMap：先执行map，再扁平化（合并成一个列表）示例：rdd = sc.parallelize([2,3,4])rdd2 = rdd.map(lambda x: range(1,x))rdd2.collect()map结果：[[1], [1, 2], [1, 2, 3]]rdd1 = rdd.flatMap(lambda x: range(1,x))rdd1.collect()flatMap结果：[1, 1, 2, 1, 2, 3] 3、mapPartitions：先将列表进行分区，再把每个分区作为一个整体处理示例：rdd = sc.parallelize([1, 2, 3, 4], 2) #把1234分成两个分区，一个是12，一个是34def f(iterator): yield sum(iterator) #定义一个函数，传入一个迭代类型，进行求和rdd.mapPartitions(f).collect()执行结果：[3,7] 即两个分区的求和结果

4、mapPartitionsWithIndex：在mapPartitions基础上，传入的函数可接收两个参数，第一个参数为分区编号，第二个为对应分区的元素组成的迭代器示例：rdd = sc.parallelize([1, 2, 3, 4], 4) #分成4个分区def f(splitIndex, iterator): yield splitIndex #函数直接返回分区编号rdd.mapPartitionsWithIndex(f).collect()结果：[0, 1, 2, 3] 即各个分区的编号
5、filter：用于过滤。接受一个函数作为参数，对每个元素判断，结果为True的保留，否则去除示例：rdd = sc.parallelize([1, 2, 3, 4, 5]) rdd.filter(lambda x: x % 2 == 0).collect() #保留偶数结果：[2, 4] 6、distinct：去重示例：rdd = sc.parallelize([1, 1, 2, 3])rdd.distinct().collect()结果：[2， 1， 3] 7、union：求两个rdd并集，不去重，接受另一个rdd作为参数示例：rdd1= sc.parallelize([1,2,3])rdd2= sc.parallelize([4,5,6])
rdd1.union(rdd2).collect()结果：[1,2,3,4,5,6] 8、intersection：求两个rdd的交集，去重示例：rdd1 = sc.parallelize([1, 10, 2, 3, 4, 5])
rdd2 = sc.parallelize([1, 6, 2, 3, 7, 8])
rdd1.intersection(rdd2).collect()结果：[1, 2, 3] 9、subtract：返回rdd1有，rdd2没有的元素，且不去重示例：rdd1 = sc.parallelize([1, 2, 2, 3, 4, 5])
rdd2 = sc.parallelize([1, 3, 5])
rdd1.subtract(rdd2).collect()结果：[2 , 2, 4] 10、sortByKey：根据key进行排序示例：tmp = [(‘a’,1), (‘b’,2), (‘1’,3), (‘d’,4), (‘2’, 5)]
sc.parallelize(tmp). sortByKey()
结果：[(‘1’, 3), (‘2’, 5), (‘a’, 1), (‘b’, 2), (‘d’, 4)]*默认升序，如果需要降序，加参数ascending=True 11、sortBy：根据指定key进行排序，key由一个函数获取示例：tmp = [(‘a’,1), (‘b’,2), (‘1’,3), (‘d’,4), (‘2’, 5)]

rdd.mapValues(f).collect()

6、两种共享变量的含义和作用

累加器在Driver节点创建并初始化，然后在Worker节点中的Executor更新，最后回到Driver端读取最后的值解决方法：Spark提供累加器（accumulator）共享变量代码修改如下：num = sc.accumulator(0) def func(x): global num num += x rdd = sc.parallelize(range(10))rdd.foreach(func)print(num) #结果为45 广播变量示例：b = sc.broadcast([1, 2, 3, 4, 5]) #将一个通用数据设为广播变量b.value #查看广播变量的值打印结果：[1, 2, 3, 4, 5]

广播变量总结：如果executor端用到了Driver的变量，在不使用广播变量的情况下，executor有多少个task就要复制多少个变量副本，若使用广播变量，每个executor只需一个变量副本广播变量只能在Driver端定义不能将一个rdd设为广播变量广播变量的值只能在Driver端修改

共享变量总结：相当于编程学里的全局变量如果这个全局变量在使用过程中需要累加，则设为累加器如果这个全局变量全程不需要改变，则设为广播变量

广播变量适用于保存共享配置文件、通用的数据，常用的数据结构等，不适合存放太大的数据广播变量不会内存溢出，因为其数据的保存的 Storage Level 是 MEMORY_AND_DISK 的方式

7、宽依赖、窄依赖

窄依赖：是指每个父RDD的数据块最多被一个子RDD依赖。（一对一或多对一）比如map、filter、union等； 宽依赖：是指一个父RDD的数据块会被多个子RDD依赖；（一对多）比如groupByKey、reduceByKey、 sortByKey等； 宽依赖、窄依赖的简单区分：要不要进行“洗牌”操作如map、filter等，输入输出数据的过程不需要打乱重组，属于窄依赖如groupByKey，reduceByKey等需要打乱分组，属于宽依赖

DAG（有向无环图）的生成 原始的RDD通过一系列的转换就形成了DAG，在spark里每一个操作生成一个RDD，RDD之间连一条边，最后这些RDD和他们之间的边组成一个有向无环图。

8、stage划分方法

目的：每个stage内的任务执行不需要等待其他stage，提高计算效率Stage划分方法1、将所有任务圈为一个stage2、从最后一个rdd向前找，遇到宽依赖，则新增一个stage圈，若遇到窄依赖，则加入当前圈3、最终划分stage个数 = 宽依赖个数 + 1

9、RDD持久化方式

与存储的区别：存储的目的是以文本形式保存RDD结果到磁盘中，用于后续分析持久化的目的是RDD的重用，可以灵活指定存在内存还是磁盘中

1、图的概念

图：(Graph)是由顶点的非空有限集和边的有限集构成的，记作G=<V,E>

度：与顶点连接的边的数量，若是有向图，分出度和入度

路径和连通：

连接两个不同顶点的序列为路径，序列中不含重复顶点

若两顶点之间存在路径（能从一个点走到另一个点），则称两个顶点连通，任意两个顶点均连通，则称G是连通图，若有向图中，任意两个顶点互相都连通，则称为强连通图
